# 后端方向约束

## 后端角色

Java 后端是 Farm 项目的权威业务结算层，负责玩家、土地、作物、仓库、摆摊、金币、排行榜、广告奖励和配置下发。

客户端可以负责表现和操作，但核心资产变更必须由后端校验和落库。

## 开发前必须确认

每次修改 Java 后端前，先确认：

1. 是否影响现有 Cocos 客户端调用。
2. 是否需要更新 Swagger/OpenAPI。
3. 是否需要更新 DTO 字段说明。
4. 是否需要更新实体字段注释、表注释或 SQL 初始化脚本。
5. 是否需要更新业务模块文档、MVP 进度文档和开发日志。

## 模块方向

| 模块 | 方向约束 |
| --- | --- |
| user | 登录和玩家档案必须稳定，开发期 token 简单但语义清晰 |
| farm | 土地、播种、成熟、收获是核心闭环，不允许只做展示不做校验 |
| inventory | 仓库承接农场产出，后续服务摆摊食材消耗 |
| stall | 摆摊是下一阶段核心玩法，后端负责开局、订单、结算和收益校验 |
| economy | 金币变化必须可追溯，后续所有收益要写流水 |
| gameconfig | 作物、菜谱、升级、事件优先配置化，不把数值散落在代码里 |
| ad | 广告奖励必须幂等，不能让客户端重复领取 |
| leaderboard | 排行榜以后以后端统计为准，不信任客户端提交分数 |

## 接口方向

- 接口要服务 Cocos 客户端实际玩法，不只服务 Apifox 测试。
- 响应字段要考虑前端渲染需要，例如土地状态、成熟倒计时、作物名称、库存数量。
- Java `Long` ID 对浏览器和 Cocos Web 有精度风险，对外 JSON 必须以字符串表达。
- 接口新增或修改后必须同步 Swagger 注解、`docs/API.md` 和 `docs/BUSINESS_MODULES.md`。

## 数据库方向

- 数据库结构必须可重建、可迁移、可说明。
- 表和字段要有注释。
- 初始化 SQL、Flyway migration、数据库字典文档必须保持一致。
- 已经提交的 Flyway migration 原则上不修改，后续结构变更新增版本脚本。

## 日志和回退

每次后端任务完成后，必须追加 `logs/CHANGELOG.md`，并写清：

- 完成内容
- 修改文件
- 验证结果
- 回退建议

没有验证成功时，也要明确写出未验证原因。
